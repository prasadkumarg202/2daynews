tap2news Design Audit · July 2026

The optimized tap2news: what it looks like, and the gaps between here and there

North star: a reader opens the app to their district in their script in under 2 seconds; every scroll answers "what does this mean for my town"; every fifth card asks them to act; the whole machine runs on ₹0 fixed API cost + one Gemini call per story. We are ~80% there. The 20% below, in priority order.

What's already strong — don't touch
✓Distinct brand: vermillion + ink + warm paper, Anek/Noto Indic-first type, category color-coding — not a Dailyhunt clone
✓Signature interactions: Quick Read swipe FAB, breaking ticker, engagement cards (polls/quizzes), GPS "మీ దగ్గర"
✓Admin v2 command center matches the real backend (queue workers, Redis, cron, cost console)
✓Cost architecture is coherent end-to-end: local data > cached APIs > paid gap-fillers > Gemini as the only scaling cost
Mobile app — reader experience
P1
Real imagery in NewsCards

The T2 monogram placeholder is the single biggest "not real yet" signal. Wire the image pipeline's WebP output (or 6–8 sample WebPs now) into hero/row cards; add a 1.5s shimmer skeleton for loading.

low effort
P1
Dark mode

Night reading is prime time for news apps in India. The ink scale is already dark-ready — add a `data-theme="dark"` token set (paper→ink-950, cards→ink-900) and a toggle in the header. ~30 token overrides, entire app follows.

low effort
P2
Text-size control

Tier-2/3 readers skew older; a 3-step size control (A− A A+) in the header persisting to localStorage is the most-requested vernacular-app feature.

low
P2
Viral Shorts vertical player

The shorts tab routes to Quick Read today. A dedicated vertical video reel (YouTube Shorts ingest is already scheduled every 30 min in cron) closes the Dailyhunt/Josh gap.

medium
P3
Offline & data-saver

Cache last 40 stories + images for offline commute reading; a data-saver toggle that skips images. Big retention lever on 2G/3G districts.

medium (production)
Admin console
P1

Two admin kits now overlap (v1's 12 screens vs v2's hubs). Port v1's Analytics/Push/SEO/Users detail into v2's Growth hub as tabs, then retire v1 — one source of truth.

Pipeline live view

Admin v2 shows queue counts; add the per-story trace (ingest_jobs stages as a horizontal stepper) so an editor can see where any story is stuck — the doc describes it, the UI doesn't show it yet.

medium

Image library manager

A screen for the image_assets library (upload, tag entities/keywords, see least-recently-used) — the operational face of the zero-cost thumbnail plan.
