tap2news Zero-API Cost Architecture

Own the data once, serve it forever

Every recurring per-request API is replaced by a one-time free-dataset import into Postgres, computed locally, or cached in Redis for all users. MCP agents replace the human ops loop. Gemini rewriting stays the only meaningful per-unit cost.

01 · Places DB replaces Google Maps

All Indian places, streets & pincodes — in your own Postgres

One-time import of free, open datasets. Every lookup that would hit Google Places/Geocoding becomes a local indexed query — ₹0 per call, works offline, Telugu names included.

Free source datasets (import once)

• OpenStreetMap India extract — streets, POIs, villages (ODbL, free)
• Census towns & villages directory — every village with district/mandal codes
• India Post pincode directory — open government data, all offices + lat/lng
• Datameet district/state shapefiles — polygons for PostGIS geo-fencing (already used by region_for_point)

Schema & lookup

places(id, name_en, name_te, kind,     -- village|town|street|poi
  mandal, district, state, pincode,
  location geography(point), tz text)
-- autocomplete: pg_trgm index on (name_en, name_te)
-- reverse geocode: PostGIS point-in-polygon
-- birth-place picker: same table, kind='village|town'

Serving: autocomplete via ilike/pg_trgm (<10ms), hot districts pinned in Redis. The app's location picker, hyperlocal "మీ దగ్గర" section, and citizen-report geotagging all read this table — no Maps SDK anywhere.

02 · Astrology computed locally

Birth place from places DB, panchang from ephemeris math

Birth-place lookup

User types their village → local places autocomplete returns lat/lng + timezone. No geocoding API. Works for 600K+ villages in Telugu script.

Panchang engine

Tithi, nakshatra, rahu kalam, muhurtham are astronomy, not content — computed with an ephemeris library (Swiss Ephemeris / astronomia) from lat/lng + date. Zero API, exact for any village.

Serve & cache

Daily panchang computed once per district at 4am → Redis (24h TTL). Horoscope text: one Gemini batch call for 12 rasis/day — 12 calls/day total, not per user.

03 · Local image library replaces image APIs & AI generation

Every politician, hero & city — tagged once, reused forever

News thumbnails are mostly the same recurring entities. Instead of per-article image search or AI generation, build an image_assets library from free/openly-licensed sources and compose thumbnails locally with Sharp.

The library (one-time + drip updates)

• Politicians: Modi, Amit Shah, Chandrababu Naidu, Pawan Kalyan, Stalin, KCR, Revanth Reddy…
• Film stars: Amitabh, Chiranjeevi, Vijay, Kamal Haasan, NTR, Allu Arjun…
• Places: Hyderabad, Vijayawada, Mumbai, Delhi landmarks, assembly buildings…
• Stock scenes: courts, rain/floods, cricket stadiums, gold, markets, exam halls
Sources: Wikimedia Commons (PD/CC — store license + attribution per asset), government press-photo releases, own correspondent photos. 2–3K curated assets cover ~80% of stories.

Schema & matching

image_assets(id, entity,            -- 'modi', 'chiranjeevi', 'hyderabad'
  kind,                -- person|place|scene
  keywords text[],     -- ['మోదీ','PM','BJP','ప్రధాని']
  variants jsonb,      -- portrait|waving|rally|closeup
  url, license, attribution, last_used)

Worker C already extracts tags/keywords per story → match against keywords[] (Telugu + English) → pick least-recently-used variant so feeds don't repeat the same photo.

Composition, not generation

Sharp composes the thumbnail locally: entity photo + place/scene background + category color band + headline strip → 900×500 WebP q82 → news-images bucket. Two entities in one story (e.g. Modi + Chandrababu) = side-by-side composite. Cost: CPU only, ~₹0.

New fallback order

1. Article's own image (RSS/og:image)
2. Local image_assets match + compose
3. Wikimedia Commons live search
4. Unsplash category preset
5. AI generation — last resort only, and the output is saved back into image_assets so it's never generated twice.

One-line breaking flow

Editor types one line in the Breaking desk ("విజయవాడలో అగ్నిప్రమాదం") → Fetch/Brave-Search MCP pulls 3–5 corroborating sources → Gemini rewrites with the standard master prompt + fact list → entities extracted → thumbnail composed from the local library → publish + push. Human effort: one sentence. API cost: one Gemini call.

Live cricket — recommended sourcing

Primary · CricketData.org (CricAPI)

Lifetime free tier (100 hits/day) to build on; paid from $5.99/mo (2K hits/day) to $29.99/mo (100K/day). Live scores, ball-by-ball, scorecards, player stats. Our 8s shared Redis key needs ~11K hits/match-day → the $12.99 tier covers a full IPL day.

Commentary · Cricbuzz API (API.market)

Real-time Cricbuzz data: commentary, news, ICC rankings, IPL fixtures. 7-day free trial, then $19.99/mo (30K units) / $59.99 (120K). Use for rich match pages + Telugu commentary rewrites, not the score ticker.

Scale-up · EntitySport / Roanuz

Official-grade ball-by-ball push feeds when DAU justifies it ($150–210/mo). Skip until IPL traffic proves the need. Avoid unofficial "free Cricbuzz" scraper APIs in production — ToS + reliability risk.

Serving: worker polls once per 8s per live match → Redis cricket:live:<match> → all users read the cache. Score widget, Quick Read cards, and push alerts (wickets, 50s, results) all consume the same key — API cost is per match, never per user.

07 · MCP agents replace human intervention

The ops team is a Claude Code loop

Supabase MCP · data ops

Agent seeds/updates the places DB, runs migrations, fixes RLS, prunes archives — "import the new pincode file" is one prompt, not a DBA ticket.

Fetch + Brave Search MCP · source ops

Nightly agent validates all 50+ RSS feeds, finds replacement URLs for dead ones, and fact-checks stories the fake-score gate flagged — shrinking the human review queue.

PostgreSQL MCP · perf ops

Weekly agent reads slow-query stats, proposes/creates indexes, reports cache hit-rate drift before it becomes an outage.

Filesystem + GitHub MCP · code ops

Dead-letter queue errors → agent reads pipeline_failures, patches the parser, opens a PR with the fix and test.

Playwright MCP · QA ops

Every deploy: agent opens the app, switches languages, taps a story, votes a poll, screenshots — regressions caught without a QA engineer.

Context7 MCP · dev ops

Agent codes against current Next.js/Supabase/Tailwind docs — fewer broken builds from stale API knowledge.
